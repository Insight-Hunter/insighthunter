PLATFORM OVERVIEW
======================================================================

InsightHunter is a multi-tenant AI CFO SaaS running entirely on Cloudflare's edge. Every tenant (org) is isolated by org_id at the database query level. Enterprise tenants additionally get their own isolated Worker runtime via Workers for Platforms. All authentication flows through a single internal worker called insighthunter-auth. Every other worker validates tokens either by calling insighthunter-auth's internal /auth/verify endpoint using an X-Internal-Secret header, or by verifying the JWT signature locally using the shared JWT_SECRET.

AUTH CLIENT  --  packages/@ih/auth-client/src/index.ts
======================================================================

Use only Web Crypto API. No external JWT libraries. Implement HMAC-SHA256.

Export async function signJWT(payload without iat/exp, secret: string, ttlSeconds: number): Promise<string>
  Encodes header as base64url JSON with alg HS256 typ JWT
  Encodes payload as base64url JSON adding iat as now and exp as now plus ttl
  Signs header.payload with HMAC-SHA256 using secret
  Returns header.payload.signature all base64url encoded

Export async function verifyJWT(token: string, secret: string): Promise<JWTPayload>
  Splits on dot into three parts
  Verifies HMAC signature
  Throws if signature invalid or exp is in the past
  Returns decoded JWTPayload

Export function extractBearer(authHeader: string or null or undefined): string or null
  Returns token string if header starts with Bearer, otherwise null

Export function jwtToAuthUser(payload: JWTPayload): AuthUser

--- API Routes: src/index.ts ---

Use Hono. All CORS headers must include Authorization, Content-Type, X-Internal-Secret.
Allowed origins: https://insighthunter.app and any subdomain.

  POST /auth/register
    Body: email, password (min 8 chars), name, org_name
    Creates org row then user row in D1 batch
    Password stored as HMAC-SHA256(plain + salt) colon salt
    Returns: access_token, refresh_token, user object

    Issues new access token

  POST /auth/logout

    Returns: message logged out

--- src/index.ts logic ---

The dispatch worker is the single entry point for all API traffic from the
frontend. It inspects the Bearer token, calls AUTH_WORKER internally to
verify it, reads the org's tier and worker_script from the auth response,
then routes to the correct downstream worker.

Routing logic:
  Extract token from Authorization header
  Call AUTH_WORKER fetch POST /auth/verify with X-Internal-Secret and body token
  If 401 return 401 to client
  Parse AuthUser from response
  Check if org has worker_script set (enterprise custom worker)
  If enterprise and worker_script exists:
    Get tenant worker from TENANT_WORKERS dispatch namespace using worker_script name
    Forward entire request to tenant worker with X-IH-User header containing JSON AuthUser
  If not enterprise or no custom worker:
    Route by URL path prefix:
      /api/bookkeeping/* → BOOKKEEPING_WORKER
      /api/bizforma/*    → BIZFORMA_WORKER
      /api/pbx/*         → PBX_WORKER
      /api/payroll/*     → PAYROLL_WORKER
      /api/ai/*          → AI_WORKER
    Strip the /api/worker-name prefix before forwarding
    Inject X-IH-User header with JSON AuthUser so downstream workers trust it
    Inject X-Internal-Secret header so downstream workers know request came from dispatch

Rate limiting per tier using KV:
  Key: rate:orgId:minute (value = count, TTL = 60 seconds)
  Free: 20/min, Lite: 60/min, Standard: 120/min, Pro: 300/min, Enterprise: null
  Return 429 with Retry-After if exceeded

--- Tenant Worker Provisioning API ---

  POST /admin/tenants/:orgId/worker
    INTERNAL ONLY X-Internal-Secret required
    Body: script_name (the worker name to register in the namespace)
    Calls Cloudflare REST API to upload the base tenant worker script
    URL: https://api.cloudflare.com/client/v4/accounts/ACCOUNT_ID/workers/dispatch/namespaces/insighthunter-tenants/scripts/SCRIPT_NAME
    Authorization: Bearer CF_API_TOKEN
    After upload calls AUTH_WORKER PATCH /auth/org to set worker_script
    Returns: deployed true, script_name

  DELETE /admin/tenants/:orgId/worker
    Deletes tenant worker from dispatch namespace
    Clears worker_script in auth

======================================================================
CODING STANDARDS — ENFORCE THROUGHOUT
======================================================================

1. TypeScript everywhere. No any except when unavoidable, use unknown.
2. ES Modules exclusively. No CommonJS require.
3. Hono for all Worker routing. Never use raw addEventListener fetch handler style.
4. D1 batch() for all multi-statement writes. Never loop with sequential awaits.
5. Org isolation: every single D1 query must include WHERE org_id = orgId.
6. Auth in sub-workers: read X-IH-User header injected by dispatch, parse JSON, trust it.
   Never re-verify JWT in sub-workers — dispatch already did it.
7. Internal calls between workers use X-Internal-Secret header matching JWT_SECRET.
8. CORS: allow credentials, explicit origin whitelist, allow Authorization and Content-Type.
9. Error responses always return JSON with error string key and appropriate HTTP status.
10. Rate limiting enforced in dispatch worker only. Sub-workers trust dispatch did it.
11. Analytics Engine writeDataPoint is never awaited. It is fire-and-forget.
12. Tier gates in sub-workers: read tier from X-IH-User header, check against TIER_LIMITS.
    Return 403 with error message that includes required tier.
13. Svelte components use createEventDispatcher for parent communication.
14. Astro pages do all session checks via middleware. Never check auth client-side.
15. All CSS uses CSS custom properties from theme.scss. No hardcoded hex values.
16. Sandtaupe theme: sand 50-900 scale and taupe 50-900 scale.
    Primary action color is sand-600. Text color is taupe-800.
